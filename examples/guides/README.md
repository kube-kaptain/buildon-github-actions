# Guide Usage Guide :-D

Each guide has the following three things in it:

* `README.md` - describing what it does ane why you'd want it how to use the system for this purpose
* `build.yaml` - minimal ref to the correct workflow
* `KaptainPM.yaml` - configuration to achieve the desired outcome

It is expected that you take the `build.yaml` and place it in `.github/workflows`
in your project and take the `KaptainPM.yaml` and drop it in your project root.

Customise as you wish.
